# Implementation Plan: Admin Dashboard

**Branch**: `001-admin-dashboard` | **Date**: 2026-06-10 | **Spec**: specs/001-admin-dashboard/spec.md

## Summary

Add an admin dashboard under `/admin/*` routes to the existing Next.js blog. Features: GitHub OAuth login, overview stats, posts table, keyword queue, SEO analytics, deployment status, and action triggers. All data sourced from GitHub API, Vercel API, and Supabase. All write operations via GitHub Actions workflow_dispatch.

## Technical Context

**Language/Version**: JavaScript/TypeScript, Node 18+, Next.js 14.2.35

**Primary Dependencies**:
- next-auth@4 (NextAuth.js with GitHub OAuth)
- recharts (charts)
- lucide-react (icons — already installed)
- @radix-ui/* (shadcn/ui primitives — already installed)
- shadcn/ui components (Button, Card, Badge, Table, Dialog — already installed or installable via CLI)

**Storage**: No new storage. Data fetched live from:
- GitHub API (posts, queue) — via `fetch()` in Server Components
- Vercel API (deployments) — via API route proxies
- Supabase (GSC analytics) — via `@supabase/supabase-js`

**Testing**: Manual testing via `npm run dev`, visual verification in browser

**Target Platform**: Web (Vercel, same project)

**Project Type**: Web application (routes added to existing Next.js app)

**Performance Goals**: Pages load in under 3s. API response caching with 5-minute TTL.

**Constraints**: No additional backend services. All data from existing external APIs.

**Scale/Scope**: Personal admin dashboard for single blog (~200 posts)
